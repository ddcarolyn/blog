1. 一句话结论

Keychron 现售 702 个产品，但真正的"键盘"只有 336 个，其余是键帽、轴体、配件、鼠标。它的官方分类（product_type）是混乱的——把键盘按"形态/连接/是否客制"三种不互斥的维度乱拆，还有 134 个塞进 Z-Others 垃圾桶、25 个完全空白。我用「一级品类 → 键盘产品线家族 → 决策分面」三层重构，让一个小白能沿着"我是谁、我要什么"找到键盘，也让运营看清每条产品线的定位与健康度。

4. 为什么这么分类（背后的思考）

4.1 先说官方分类错在哪

Keychron 自带的 product_type 是这样的：Custom Keyboards 230 / Z-Others 134 / Keycaps 123 / Switches 72 / Slim Keyboards 60 / Wireless Keyboards 28 / (空白) 25 / Mice 19 / Wired Keyboards 11。三个硬伤：

  • 维度打架，不互斥（non-MECE）。"Custom / Slim / Wireless / Wired"四个桶，分别按"是否客制 / 形态 / 连接 / 连接"切——一把无线的超薄客制键盘该进哪个？逻辑上四个都对，于是分类失去意义。
  • 垃圾桶 + 空白。134 个进 Z-Others、25 个完全没填。这意味着近 1/4 的目录在官方分类里"无家可归"，站内导航和筛选自然会漏掉它们。
  • 对消费者无指导性。"Custom Keyboards"对小白毫无意义——他不知道 custom 是贵还是便宜、是不是要自己装。

4.2 我的分类遵循三个原则

  • 第一层 MECE、互斥可穷尽。先用"是不是键盘整机"把目录干净切成 6 类，避免配件混进键盘流。
  • 第二层贴合 Keychron 真实产品逻辑（字母系列 = 价格×做工×固件阶梯）。这是它真正的"产品线"，运营对齐内部话术，消费者也能记住"K 走量、Q 旗舰、V 平价客制、B 超薄、Lemokey 游戏"。
  • 第三层用"决策分面"而非属性堆砌。布局/连接/热插拔/磁轴/价位，是消费者下单前真正要拍板的几个开关——把它们做成筛选维度，而不是埋在标签里。

一句话：官方分类是"仓库视角"（按它怎么入库），我的分类是"买家视角 + 产品线视角"（按人怎么挑、按线怎么管）。

6. 对你（运营总监）的即时信号

  • 高 C 系列售罄 8/11、整体售罄率 32%。要么是主动淡出有线入门线，要么是备货/下架流程没跟上。第一周值得拉一张"售罄但仍可见"清单（数据库已附"售罄清单"sheet）确认是策略还是事故。
  • 高 分类与数据卫生。Z-Others 134 + 空白 25 + $17,880 脏价，直接影响站内筛选、推荐、广告 feed 质量。重建 product_type/分面是高 ROI 的上手项。
  • 中 磁轴 HE 只有 50/336。这是行业增长最快的方向（见机械键盘调研 6.4），竞品 Wooting 在电竞侧紧逼。HE 线的货盘宽度与站内权重值得评估。
  • 中 QMK/VIA 已是标配（283/336）、热插拔约 280。这两个曾是差异化卖点，如今是入场券——叙事重心应转向磁轴、Mac 生态、人体工学等仍有差异的点。
