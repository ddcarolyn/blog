# 现象（可点击验证） 为什么可能是问题 想确认
1 best-keyboard-for-mac 页面 404：https://www.keychron.com/blogs/news/best-keyboard-for-mac Keychron 是 Mac 起家，"best keyboard for mac"是转化最高的词之一，落地页却 404；而 best-keyboard-for-programming / for-work 这些都在 是已知吗？是被删了还是从没建？能不能补？
2 过期大促页仍 live 且可被搜索引擎收录：https://www.keychron.com/collections/prime-day-sale-2025（2026 年仍返回 150+ 产品）；sitemap 里还有 prime-day-2022、bfcm-2024 等 和常青集合内容重复，分散 SEO 权重、浪费爬虫预算 是有意保留，还是活动结束后没清理？
3 首页 0 个 hreflang，但 keychron.uk 是独立站：https://www.keychron.com/（查源码无 hreflang）+ https://www.keychron.uk/（存在） 多区域站点没有 hreflang，Google 不知道把哪个站给哪个国家，有重复内容 / 串站风险 多站点是怎么规划的？hreflang 是已知缺口吗？
4 产品页结构化数据缺 aggregateRating：https://www.keychron.com/products/keychron-k2-pro-qmk-via-wireless-mechanical-keyboard（用 Google Rich Results Test 贴这个链接即可验证，服务端 JSON-LD 里无评分） 页面上有 Judge.me 评价，但搜索结果里拿不到星级富媒体，白白少了点击率 是已知吗？能不能把评分注入 Product schema？
5 售罄率异常高：抓 https://www.keychron.com/products.json 得 702 个在售里 221 售罄（31%）；同方法抓竞品，Epomaker 10%、Glorious 2.4%、Qwertykeys 3% 售罄率比竞品高一个数量级，可能是停产没下架，或备货 / 上下架流程问题 这是策略（清库 / 停产）还是事故？
6 一个起售价 $17,880 的异常 SKU（在 products.json / 我做的产品数据库里，落在 V 系列） 几乎肯定是批发 / 展示价错挂，会污染站内价格筛选和广告 feed 是不是脏数据？该不该清掉？
7 官方分类（product_type）混乱：134 个产品塞进 Z-Others、25 个完全空白；键盘按 Custom / Slim / Wireless / Wired 四种不互斥的维度分（数据见我做的产品数据库.xlsx） 分类不 MECE，影响站内筛选、推荐、广告 feed 质量 是历史遗留吗？有没有重构计划？
8 collection / page 体量失控：sitemap 显示 546 个 collection、1302 个 page，存在大量近义重复（wireless-keyboards vs bluetooth-keyboard vs 2-4ghz-wireless-keyboards）和无内链的孤儿页 thin / 重复内容拖累 SEO 和爬虫预算 是有意的长尾策略，还是缺治理？
